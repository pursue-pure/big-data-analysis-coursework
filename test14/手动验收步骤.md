# 实验十四手动验收步骤

## 1. 一键启动与前端页面截图

在项目根目录运行：

```powershell
cd test14
..\data_env\Scripts\python.exe run_app.py
```

如果默认端口被占用，可指定端口：

```powershell
..\data_env\Scripts\python.exe run_app.py --port 8020
```

## 6. 全新虚拟环境依赖重建截图

如果当前 Python 的 `venv/ensurepip` 可用，运行：

```powershell
cd test14
python -m venv test_env
.\test_env\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python -c "import fastapi, uvicorn, pandas, duckdb, httpx; print('clean_env_import_ok')"
```

截图保存为：

```text
test14/images/screenshot_clean_env_install.png
```

注意：本次自动尝试时，当前 Python 创建的新环境没有 `pip`，因此该步骤需要你在可用 Python 环境中手动完成。

## 7. Git 提交与远程仓库截图

建议提交信息：

```text
feat(m4): integrate week14 e2e startup, dependency cleanup and explicit degradation guard
```

执行前先检查不要提交大文件：

```powershell
git status
git add test14/README.md test14/requirements.txt test14/run_app.py test14/dashboard test14/flowchart_week14.mmd test14/.gitignore test14/.env.example test14/实验十四_M4系统联调与工程规范实验报告.md test14/手动验收步骤.md
git commit -m "feat(m4): integrate week14 e2e startup, dependency cleanup and explicit degradation guard"

```
